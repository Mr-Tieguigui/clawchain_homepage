Product

A control plane around risky agent execution, not a passive log collector.

ClawChain is built for the moment where an operator needs to stop trusting blind execution. It keeps the live session, the recovery material, and the proof trail in one coherent runtime path.

Monitor live sessions

Surface the coding-agent sessions already running on the host and route the right one into controlled execution.

Recover destructive actions

Preserve snapshot-backed recovery material before delete-style operations become permanent loss.

Verify the trail

Export readable proof logs with receipts, submissions, impact sets, and chain-verifiable anchor fields.

Quick Start

Bring the system up, then run the first controlled loop.

Setup initializes config, starts the runtime service, and launches the UI. The first operator loop is then straightforward: Join Monitor, perform the risky action, restore, export proof, and verify.

Windows 
conda create -y -n ClawChain python=3.12 pip
conda activate ClawChain
pip install -r ../ClawChain/requirements.txt
pip install -e ../ClawChain
../ClawChain/setup_clawchain.cmd 8888
Linux / macOS 
conda create -y -n ClawChain python=3.12 pip
conda activate ClawChain
pip install -r ../ClawChain/requirements.txt
pip install -e ../ClawChain
bash ../ClawChain/setup_clawchain.sh 8888
First operator loop 
1. Start or find a live coding-agent session
2. Open the UI
3. Click Join Monitor
4. Trigger a delete-style action
5. Restore the target
6. Export proof
7. Verify EVM fields
