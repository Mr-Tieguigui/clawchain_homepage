ClawChain

Runtime safety layer for autonomous coding agents.

A control surface for high-privilege agent sessions: discover what is live, route execution into a monitored path, preserve recovery material before destructive loss lands, and keep a proof trail strong enough to verify.

Quick Start See Session Detail

Product Intro

Built for the moment when an agent has enough privilege to break real things.

ClawChain is not a passive logger. It is an operator-facing runtime that sits on the path of live coding-agent work, captures destructive intent, keeps recovery material close, and gives teams a clean way to inspect, restore, export, and verify what actually happened.

Live Control

Discover and route active sessions without rebuilding the workflow from scratch.

Recovery First

Preserve snapshots before deletes, moves, and integrity mutations turn into irreversible loss.

Proof Path

Export verifiable evidence and anchor the recovery trail against the chain backend when enabled.

Quick Start

Bring the runtime up, then enter the first controlled loop.

Clone and install once, launch the runtime, then choose the setup path that matches the host OS you are operating on.

1

Clone & Install

git clone <your-repo>
cd ClawChain
pip install -r requirements.txt
pip install -e .
2

Start Project

Launch the setup entrypoint
Wait for service-start + UI health
Open the operator console
Join Monitor on the active session
Windows
conda create -y -n ClawChain python=3.12 pip
conda activate ClawChain
setup_clawchain.cmd 8888
Linux
python3 -m venv .venv
source .venv/bin/activate
bash setup_clawchain.sh 8888
macOS
python3 -m venv .venv
source .venv/bin/activate
bash setup_clawchain.sh 8888

Control Sequence

From detection to verified recovery.

The product loop is designed so the operator can move from live discovery to proof-backed intervention without leaving the control surface.

01

Session Discovery

Find live high-privilege agent sessions already running on the host.

02

Controlled Routing

Switch the session into a monitored launcher path with resume and handoff commands.

03

Risk Capture

Detect destructive operations early enough to keep recovery material intact.

04

Snapshot Restore

Recover the affected path from the vault and validate the restored target.

05

Proof and Anchor

Export operator-readable proof and verify fields against the configured chain backend.
